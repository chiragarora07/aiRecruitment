What it does

Every stage of hiring, structured by design

No more blind interviewers or scores nobody can explain. Each piece of the pipeline does one job, enforced at the server — not the spreadsheet.

01

Role-Gated Workspaces

Admins, Applicants, and Interviewers each get a dedicated portal — and the separation isn't cosmetic. Every endpoint checks role at server-side middleware, so no client can reach data it isn't assigned to.

02

Dual-Pass AI Screening

One pass parses each CV for skills and fit. The other generates a fresh set of interview questions per candidate — never the same two, never from a static bank. Both run in the background, off the request thread.

03

Linear Pipeline Enforcer

Drives move through fixed stages — setup, screening, interviews, results — and the API enforces the order. No stage can be skipped or reopened by accident.

04

Dynamic Score Matrix

CV score, AI round score, and interview score combine using weights the admin sets per drive. Change a weight after scores are in, and every ranking recomputes instantly — no candidate falls through the cracks.

05

Result Notifications

One click sends every acceptance, waitlist, and rejection email — each templated, none manual. A delivery log shows exactly which messages landed and which didn't.

Persistent Organization Hubs

A public page that doesn't go dark between hiring seasons. Past and present drives, official announcements, and event registration — visible to anyone, no login required.
